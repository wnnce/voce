# 核心特性 (Key Features)

- **内存池化管理**：通过内存复用与引用计数（Ref-counting）机制，减少高频媒体流处理中的 GC 分配开销。
- **Copy-on-Write**：基于 ReadOnly 与 Mutable 机制实现，确保数据在节点传输中的不可变性。
- **双优先级调度**：控制信令（Signal）与媒体数据（Media）分离，确保在数据拥塞时，系统控制和信令信息仍能优先处理。
- **双协议长连接支持**：同时支持 WebSocket (Binary Custom Packet) 与 gRPC (Bidirectional Streaming) 两种实时通信方式，在浏览器端和嵌入式/IoT 设备端均能实现低开销双向流。
- **背压与暂停控制**：在数据分流与下行发送阶段实现主动丢包策略（Backpressure），防止由于慢插件或网络拥塞导致系统 OOM；支持会话级的暂停与恢复逻辑。
- **多轨隔离与取消机制**：利用 `MultiTrackPlugin` 隔离媒体与信令流，防止慢任务阻塞实时链路；内置版本号校验机制，支持在收到取消信号时毫秒级取消活跃任务并自动清理陈旧数据。
- **配置即 Schema**：插件支持基于 JSON Schema 的配置定义，前端工作台根据 Schema 自动渲染动态表单，实现零开发配置注入。
- **声明式 DAG 编排**：支持通过 JSON 定义复杂的处理链路，节点间完全解耦，通过契约命中的校验方式进行连接。
- **完善的测试机制**：内置 `PluginTester` 与 `MockFlow`，支持对单个插件进行离线单元测试。
- **调试工具**：包含可视化 Web 工作台（可视化 DAG 编排）和支持简单监控面板的 TUI 对话客户端。
- **分布式网关集群 (Experimental)**：提供独立的网关接入层，支持多执行节点（Pod）水平扩展；通过一致性 Hash 实现会话路由粘连，并具备 Pod 故障感知与会话恢复能力。
